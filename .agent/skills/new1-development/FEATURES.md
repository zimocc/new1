# 🚀 功能清单 (FEATURES)

## 一、核心业务功能

### 1.1 多教材支持 (Multi-Textbook)
- 内置多种主流教材：新概念一册、小学一年级（新启航）、小学三年级（人教PEP）。
- 教材数据独立管理，切换教材时自动加载对应的课时列表和学习进度。

### 1.2 闯关式学习进度 (Progress Tracking)
- **多维度记忆**: 每个教材独立保存"当前课时"和"当前单词"进度。
- **持久化**: 使用 `LocalStorage` 存储，刷新页面或下次访问时自动续接。
- **实时统计**: 顶部展示当前教材的总课数、总词汇量以及本猫当前的通关进度（第 X 课 / 第 Y 词）。

### 1.3 交互式单词卡片
- **单词展示**: 自动缩放的单词文本 + 国际音标 + 词性自动映射（如 `n.` → `名词`）。
- **释义控制**: 
  - 手动模式：点击虚线框"揭开"中文秘密。
  - 自动模式：切换单词时直接显示中文。
- **快速导航**: 
  - 左右点击区：卡片两侧隐形热区支持快速切词，点击时播放彩色烟花粒子动效。
  - 导航栏：底部的"上一个"、"下一个"实体按钮。
  - **边界保护**: 当前单词为教材首词时隐藏"上一个"按钮，末词时隐藏"下一个"按钮。在边界触发切词时弹出提示。
- **切词过渡动画**:
  - 单词/音标/词性区域带方向性滑动过渡（左滑/右滑）。
  - "点我听发音"按钮保持静止，不参与过渡动画。
  - 中文释义区仅在显示状态（手动点开或自动发音模式）下参与滑动过渡。
  - **实现要点**: 使用 `slideMeaning` ref 在切词前快照释义可见状态，避免 watcher 重置 `showMeaning` 后 Transition 无法正确判定。

## 二、智能辅助功能

### 2.1 循环听写模式 (Dictation Mode) ★
专门为课后听写设计的全自动模式：
- **逻辑**: 每个单词自动朗读 **3 遍**。
- **间隔**: 每遍朗读之间停顿 **2 秒**，方便书写。
- **切换**: 念完 3 遍后中场休息 **5 秒**，自动跳转至下一个单词。
- **状态流**: 开启听写会强制关闭"自动发音"并重置相关状态。

### 2.2 自动发音 (Auto-Play)
- 开启后，点击"下一个"或"上一个"按钮时，应用会自动触发语音播报。
- 自动发音状态下，中文释义始终保持开启，适合快速预览。

### 2.3 课时列表弹窗 (Course Selection)
- 全局浮层展示所有课时。
- **智能定位**: 打开列表时，自动滚动到当前正在学习的课时位置（高亮显示）。
- **一键置顶**: 列表右下角设有悬浮按钮，快速回到顶部。

## 三、技术亮点功能

### 3.1 鲁棒的 TTS 降级链路
应用内置了 4 层语音引擎降级机制，确保在不同网络环境下都能发音：
1. **有道 (Youdao)**: 首选，对短语和特殊符号支持最好。
2. **Google Dynamic**: 备选，美音纯正。
3. **百度 (Baidu)**: 备选。
4. **Google Static (Oxford)**: 最后兜底，仅支持纯单词静态 MP3。

### 3.2 移动端体验优化
- **滚动锁定**: 弹出任何 Modal 时，禁止底层 `body` 滚动，防止误触穿透。
- **触摸响应**: 针对 WebKit 优化了 `-webkit-tap-highlight-color`，消除移动端点击阴影。
- **首次访问引导**: 针对新用户自动弹出"应用使用说明"。
- **禁用按钮点击隔离**: `.disabled-btn` 不使用 `pointer-events: none`（会导致点击穿透到底层热区），而是保留点击事件拦截，由 JS 逻辑守卫；同时禁用 `:active` 按压动画。

### 3.3 Toast 通知系统
- **章节切换提示**: 跨 unit/lesson 边界时，在章节标题位置弹出当前课时名称通知（如 `Unit 2`），1.8 秒后自动消失。
- **边界提示**: 在教材首/末单词触发越界操作时，弹出友好提示（如"已经是第一个单词啦"）。
- **定位方式**: Toast 使用 `position: absolute` 定位在 `.lesson-name-wrapper` 内，精确覆盖章节标题，不遮挡单词显示区域。
- **动效**: 弹出时使用 `cubic-bezier` 弹性缩放动画，消失时缩小淡出。

### 3.4 烟花粒子反馈
- **触发条件**: 点击卡片左/右热区切换单词时，在触摸点生成一簇彩色粒子。
- **实现方式**: 8 个随机颜色的小圆点沿随机角度向四周爆散，通过 CSS `@keyframes` 驱动位移和淡出，0.5 秒后由 JS 清理 DOM。
- **技术要点**: 通过 `event.currentTarget.closest('.main-card')` 获取卡片位置，计算相对坐标；使用 CSS 变量 (`--dx`, `--dy`, `--color`, `--size`) 传递随机参数给动画。
